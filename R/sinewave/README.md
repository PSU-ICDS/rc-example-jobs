# R Plotting Example: Noisy Sine Function

This repository provides a simple example of using R on the Roar ICDS cluster to read a dataset, process coordinates, and generate a graphical plot as a `.png` file.

## Repository Structure

* `read_nsin.R`: The R script that reads coordinate data and generates a scatter plot.
* `sub_nsin_R.submit`: A Slurm batch script configured for the ICDS cluster to execute the R script.
* `noiseysine.csv`: The input CSV file containing the raw data points.

## Analysis Overview

The R script (`read_nsin.R`) performs the following actions:
1.  **Data Ingestion**: Reads the `noiseysine.csv` file.
2.  **Data Extraction**: Assigns the first column to the X-axis and the second column to the Y-axis.
3.  **Graphic Generation**: Uses the `cairo` bitmap to create a high-quality PNG file named `r_noiseysine.png`.
4.  **Plotting**: Generates a scatter plot with the title "Noisy Sine Function".

## How to Run

Submit the job to the Slurm scheduler using:

```
sbatch sub_nsin_R.submit
```

### Monitoring and Results

Once the job is submitted, Slurm will generate several log files and an output image:

* **`nsin_R_nsin_R.<JobID>.out`**: Contains the standard output from the R session.
* **`nsin_R_nsin_R.<JobID>.err`**: Contains environment logs or R warnings.
* **`r_noiseysine.png`**: The resulting plot generated by the R script.

## Customization Tips

* **Modify Resource Directives**: Adjust `--mem-per-cpu` and `--time` in `sub_nsin_R.submit` if you are working with significantly larger datasets.
* **Update Module Path**: This script specifically uses the `module use /storage/icds/RISE/sw8/modules` path; ensure this matches your current environment requirements.
* **Plot Styling**: You can modify the `plot()` function in `read_nsin.R` to change colors, point styles, or add trend lines.
* **Output Format**: If you require a vector graphic instead of a PNG, change the `png()` function in the R script to `pdf()` or `svg()`.
